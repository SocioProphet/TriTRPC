# Codebook evolution note

This note records the current recommendation from the worldclass work:

- `topic23.proposed.v1` was the baseline
- `topic25` was the conservative minimum correction
- `topic26` is likely the stronger mature correction

## Why topic26 is likely better

The later sandbox and boundary-governance work showed that these are distinct:

1. temporal / CHRONOS-style semantics
2. environment / toolchain capability
3. boundary / artifact / export control

Earlier work compressed (2) and (3) too aggressively. The more accurate long-term direction is to keep them separate.

## Practical implication

The codebook should likely evolve in two steps:

- immediate review against current unified-v4 materials using the conservative extension posture
- explicit revisit of topic25 vs topic26 before any registry is declared canonical
